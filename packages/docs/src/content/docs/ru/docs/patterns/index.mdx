---
title: Паттерны Workflow
description: Переиспользуемые паттерны для построения workflow
---

Паттерны workflow — переиспользуемые решения типичных задач проектирования.

## Доступные паттерны

| Паттерн                                                             | Применение                                                                           |
| ------------------------------------------------------------------- | ------------------------------------------------------------------------------------ |
| [Сбор информации](/ru/docs/patterns/information-collection/)        | Сбор возможностей агента или предпочтений пользователя на старте                     |
| [Пропуск шагов](/ru/docs/patterns/skip/)                            | Возможность пропустить необязательные шаги                                           |
| [Цикл валидации](/ru/docs/patterns/validation-loop/)                | Проверка результатов и повтор при ошибке                                             |
| [Ветвление](/ru/docs/patterns/branching/)                           | Разные пути для разных сценариев                                                     |
| [Динамические файлы](/ru/docs/patterns/dynamic-files/)              | Шаблоны путей файлов для переиспользования                                           |
| [Верификация шагов](/ru/docs/patterns/step-verification/)           | Проверка завершения шага перед продолжением                                          |
| [Эскалация](/ru/docs/patterns/escalation/)                          | Обработка ошибок с участием пользователя                                             |
| [Ревью субагентом](/ru/docs/patterns/subagent-review/)              | Делегирование ревью независимым субагентам                                           |
| [Рабочее пространство](/ru/docs/patterns/workspace/)                | Организация файлов workflow в выделенной директории                                  |
| [Персистентность Notes](/ru/docs/patterns/notes-persistence/)       | Сохранение и получение данных между шагами и запусками                               |
| [Публикация Artifacts](/ru/docs/patterns/artifacts-publishing/)     | Публикация HTML контента с публичными URL                                            |
| [Статическая конфигурация](/ru/docs/patterns/static-configuration/) | Хранение инструкций как значений по умолчанию в variableRegistry для консистентности |
| [Антипаттерны](/ru/docs/patterns/anti-patterns/)                    | Типичные ошибки проектирования workflow                                              |

## Руководство по выбору паттерна

**Нужно собрать конфигурацию?** → [Сбор информации](/ru/docs/patterns/information-collection/)

**Нужны необязательные шаги?** → [Пропуск шагов](/ru/docs/patterns/skip/)

**Нужен контроль качества?** → [Цикл валидации](/ru/docs/patterns/validation-loop/)

**Нужны разные пути?** → [Ветвление](/ru/docs/patterns/branching/)

**Нужны переиспользуемые пути файлов?** → [Динамические файлы](/ru/docs/patterns/dynamic-files/)

**Нужно проверить завершение?** → [Верификация шагов](/ru/docs/patterns/step-verification/)

**Нужна обработка ошибок?** → [Эскалация](/ru/docs/patterns/escalation/)

**Нужна независимая проверка?** → [Ревью субагентом](/ru/docs/patterns/subagent-review/)

**Нужны организованные рабочие файлы?** → [Рабочее пространство](/ru/docs/patterns/workspace/)

**Нужно сохранить данные между запусками?** → [Персистентность Notes](/ru/docs/patterns/notes-persistence/)

**Нужно опубликовать отчёт с публичной ссылкой?** → [Публикация Artifacts](/ru/docs/patterns/artifacts-publishing/)

**Нужны консистентные инструкции для агента?** → [Статическая конфигурация](/ru/docs/patterns/static-configuration/)

**Нужно избежать типичных ошибок?** → [Антипаттерны](/ru/docs/patterns/anti-patterns/)
